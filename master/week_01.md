# 1️⃣ 개념 정리

## 01. 여러 가지 과제 
데이터 사이언티스트는 데이터를 활용하여 의미 있는 인사이트를 도출하고 의사결정에 도움을 주는 역할을 한다. 이를 위해서는 다양한 데이터를 이해하고 적절한 분석 방법을 선택하는 능력이 필요하다. 

AI가 빠르게 발전하고 있는 현시점에서 데이터 사이언티스트의 역할은 점점 중요해지고 있다고 생각한다. 이제는 더 이상 한 분야만 잘해서는 경쟁력을 갖출 수가 없다. 분석부터 해석까지 전반적인 흐름을 전부 파악할 수 있어야 하는 데이터 사이언티스트는 AI와 잘 융합해서 나아갈 수 있지 않을까 하는 기대가 있다. 

이렇게 나아가기 위해서 가장 중요한 과제는 당연히 기본적인 분석 능력에 대한 충족이지만, 더 나아가 팀원들과의 원활한 의사소통과 숲을 볼 수 있는 시야를 갖추는 것이라고 생각한다. 


## 02. 시스템

**PostgreSQL**
- 관계형 데이터베이스 시스템으로, 안정적인 트랜잭션 처리와 SQL 기반 데이터 관리에 강점

**Apache Hive**
- Hadoop 기반의 데이터웨어하우스 도구로, 대용량 데이터를 SQL과 유사한 방식으로 분석 가능

**Amazon Redshift**
- AWS에서 제공하는 데이터웨어하우스 서비스로, 대규모 데이터 분석을 빠르게 수행 가능

**Google BigQuery**
- 구글 클라우드 기반의 데이터 분석 플랫폼으로, 서버 관리 없이 대용량 데이터를 SQL로 분석 가능

**SparkSQL**
- Apache Spark에서 제공하는 SQL 처리 기능으로, 대규모 데이터 처리와 분석을 빠르게 수행 가능

=> *데이터 규모와 처리 방식, 실행 환경 등에 따라 적절한 도구를 선택해야 함*


### 업무 데이터 
- 정의 : 서비스나 시스템을 운영하기 위해 데이터베이스에 저장되는 데이터
- ex. 사용자 정보, 상품 정보, 주문 데이터, 리뷰 데이터 등
- 특징
  - **갱신형 데이터** : 데이터의 상태가 변경되면 새로운 데이터를 계속 추가하는 것이 아니라 기존 데이터를 수정하는 방식으로 관리
  - 정확도가 높음 : 트랜잭션과 롤백 기능을 통해 오류 방지
- 한계 : 서비스 운영을 중심으로 설계된 데이터이기 때문에 분석 목적에 맞게 추가적인 가공 必

### 로그 데이터
- 정의 : 사용자 행동이나 시스템 이벤트를 기록하기 위해 생성되는 데이터
- ex. 웹사이트 접속 기록, 클릭 로그, 페이지 이동 기록, 검색 기록
- 특징
  - **누적형 데이터** : 로그가 생성된 이후에는 기존 데이터를 수정하지 않고 새로운 데이터를 계속 추가하는 방식으로 저장
  - 과거 특정 시점의 사용자 행동을 분석하는 데 유리
- 한계 : 로그 수집 방식에 따라 데이터 누락이나 봇 트래픽이 포함될 수 있어 **정확도가 업무 데이터보다 낮을 수 있다**는 점도 고려해야함

```
💡 두 데이터를 사용해서 생성되는 가치
- 사용자가 어떤 행동을 했는지
- 그 행동이 실제 매출이나 서비스 결과에 어떤 영향을 미쳤는지

=> 함께 분석함으로써 더 깊이 있는 분석 가능
```
